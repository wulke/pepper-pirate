# EARS Specifications

Testable requirements in [EARS](https://github.com/jszmajda/lid) format with semantic IDs. These are derived from the design docs and serve as the authoritative statement of intended system behavior.

## Domains

| File | Domain | Specs |
|---|---|---|
| [GROW.md](./GROW.md) | Growing cycle — plant, tend, harvest | GROW-001–GROW-009 |
| [BREED.md](./BREED.md) | Breeding and genetics | BREED-001–BREED-014 |
| [SEASON.md](./SEASON.md) | Season cycle and prestige | SEASON-001–SEASON-020 (SEASON-010 retired) |
| [MARKET.md](./MARKET.md) | Market and trading | MARKET-001–MARKET-005 |
| [MINI.md](./MINI.md) | Mini-games | MINI-001–MINI-004 |
| [LOOP.md](./LOOP.md) | Core loop and design pillars | LOOP-001–LOOP-005 |

## Conventions

- Each spec is a single testable statement in EARS syntax.
- Status: `draft` until reviewed against design docs; `approved` once confirmed.
- `@source` links back to the authoritative design doc for the requirement.
- Spec IDs are permanent — never reuse a retired ID.
